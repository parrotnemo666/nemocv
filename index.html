我曾在聯邦銀行工作過一年,之後來到華厚又擔任後端 Java 工程師大約九個月,主要負責銀行相關的後端業務。我大學畢業的科系是食品科學。

三、學習時間安排

我已經參考過兩位學姊的直播分享,目前的安排大致是平日至少四天投入在深度學習與線性代數等基礎知識上。我認為這類知識需要長期累積與深入理解,必須和它長時間相處才能真正掌握。假日的話,週六日會挑一天專注學習,另一日則安排休息為主。另外,會規劃一日用於與同學或同儕交流,因為有時候遇到瓶頸或停滯太久會感到不舒服,而交流能讓問題更快找到解決方案。

在學習安排上,我會每週固定規劃時間投入相關知識的學習,並採用「任務分解」的方式,將複雜的主題拆解成小單元,逐步理解與消化。具體的時間規劃如下:

  • 週日晚上: 回到家後約 18:00 以後,準備隔天線上會議的相關資料與討論內容
  • 週一: 參加線上會議後,檢視學習成果,確認前一週的進度並調整本週計畫
  • 週二: 因有運動安排(打球),主要以休息與身體調整為主
  • 週三: 安排任務分配,並用 AI 開始輔助我學習當我的助教
  • 週四、週五: 下班後投入約 3 小時專心學習線性代數與相關知識。午休時間若有 90 分鐘空檔,也會抽出約 20 分鐘回顧前次學習內容
  • 週五: 與小組或同學進行小型回饋會,分享當週學習成果
  • 週六: 以休息為主,視狀況調整學習時間約 3 小時
  • 週日: 安排較長時間(約 5~6 小時)進行知識複習與整合

四、負面情緒處理經驗

我曾經在一個資策會專案的尾聲,因為進度過快而感到自己專案做不完,能力跟不上,導致情緒陷入低谷。

當時的解決方式,首先是和隊長交流,坦承自己進度較慢或成果不佳,並詢問是否有資源能協助。如果資源有限,我就會嘗試把任務拆解並量化,先完成核心必要的部分,讓專案能以「最小可行方案」運作起來,再逐步添加額外的美化或強化功能。這樣的做法讓我能逐漸從負面情緒中走出來,並確保專案能持續推進。

在過去的學習與工作過程中,我難免會遇到挫折與自我懷疑。當情緒低落時,我會透過書寫、運動或與他人交流來整理思緒,讓自己重新找回專注力。這些經驗幫助我培養了在長期挑戰中保持穩定心態的能力,也讓我更能體會持之以恆的重要性。

六、為何要學習基礎模型?

即使無法達到 OpenAI 的程度,我仍認為學習基礎模型非常重要。

本地端與遠端的需求本來就不同

OpenAI 作為商業公司,擁有龐大的資金與人力資源,我們個人或一般團隊很難達到同樣的層級。但其實我們也不需要做到那個程度,因為本地端與遠端的需求本來就不同。以銀行系統為例,若涉及重要的客戶資料,就不可能傳到遠端進行處理,因為會有資安風險,也可能受到金管會的限制。在這種情況下,本地端運行的模型雖然不一定比 OpenAI 更強大,但卻能提供安全性並解決業務需求,這就是它存在的價值。

從底層開始建構

基礎模型的學習就像建造一座金字塔,必須從最底層的架構與核心原理開始理解,才能進一步進行微調與最佳化。透過這樣的學習,我們能知道哪些參數需要調整,如何讓模型變成符合自己需求的樣子。這不僅是技術層面的收穫,也是軟體工程師的核心價值之一——理解底層原理,並將其為己所用。

另外,我曾經經營過兩年半的生意,因此對於未來的挑戰與不確定性,我始終保持開放與樂觀的態度。我認為挑戰和挫折其實正是進步的起點。在資策會受訓的六個月裡,即使我最初幾乎沒有寫過幾天程式,但仍然憑著這種態度撐過來,並完成了學習轉職的過程。

未來,我希望能加入這樣的學習社群,不僅在學習與討論中成長,我非常喜歡團隊合作,相信知識的交流能產生更大的價值。在過去的專案經驗中,我常透過討論與分享來激盪新的想法,也期許自己有一天能站在 YouTube 的直播分享中,和學弟妹們交流我自己的專案與心得。我相信這將是一個很寶貴的經驗,也是我嚮往的生活方式。我希望能藉由這樣的機會不斷精進自己,並持續回饋給這個社群。
